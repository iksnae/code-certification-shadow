# Decision journal — 23


## Decision
Language-tier capability audit complete. 22 unique language IDs from certifiableExts in internal/discovery/generic.go.

5 languages have FULL tier (parser + lint + test collectors):
- go: GoAnalyzer (go/ast), go vet + golangci-lint, go test
- ts: TSAnalyzer (tree-sitter), ESLint, Jest/Vitest
- js: TSAnalyzer (same tree-sitter grammar as ts), ESLint, Jest/Vitest
- py: PythonAnalyzer (tree-sitter), Ruff, pytest
- rs: RustAnalyzer (tree-sitter), cargo clippy, cargo test

17 languages are GENERIC tier (no parser, no lint, no test — only git history + line counts via GenericScanner):
rb, java, kt, sh, c, cpp, cs, swift, php, elixir, erlang, scala, lua, r, zig, sql, proto

No languages currently qualify for TierParse or TierLint (all languages with parsers also have lint+test; no language has lint-only without a parser).

TierForLanguage(lang) must return TierGeneric for unknown languages (fail-safe).

Test (a) must dynamically import discovery to get known language IDs — certifiableExts is unexported, so a minimal exported helper (e.g., KnownLanguages() or exporting CertifiableExts) is needed in discovery/generic.go for the test to iterate all IDs.
