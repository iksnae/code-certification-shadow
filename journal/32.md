## Decision
BLOCKER: The `Unsupported` field does not exist on `domain.CertificationRecord` or any other domain type. The issue's non-goal explicitly says "Do not modify domain types" — yet the issue description says "skip records where Unsupported == true". The parent issue #25 was supposed to add this field (per engagement history: "Grade and CertificationRecord need N/A and Unsupported additions"). The implementer must either (a) add `Unsupported bool` to `CertificationRecord` (contravening the non-goal), (b) wait for #25 to land first, or (c) detect unsupported records via an alternative mechanism (e.g., a new `StatusUnsupported` or checking whether the language has no registered analyzer). Currently there is no programmatic way to distinguish unsupported-language records from normal ones — all records are treated identically regardless of language support.
