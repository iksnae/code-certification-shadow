# Decision journal — 28


## Decision
Investigation complete. Five categories of test regressions identified across the codebase stemming from the parent issue #21 (Swift repos get fabricated grades). The core mechanism: Swift files are discovered by GenericScanner (.swift→swift) but have NO registered analysis.Analyzer (only go/ts/js/py/rs are registered via init()). The certifier's collectStructuralEvidence returns early for Swift (no analyzer, not Go), producing only file-level metrics evidence. This yields sparse dimension scores and a ~0.80 "hollow" grade. The fix from #21 is to recognize unsupported languages explicitly and exclude them from normal grading. The regressions to fix here are tests that assume all languages produce full dimension sets or non-N/A grades.
