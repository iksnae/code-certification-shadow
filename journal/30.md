# Decision journal — 30


## Decision
GradeNA must be a numeric sentinel (GradeNA Grade = -1) added in a SEPARATE const block from the existing iota block. The issue's "GradeNA Grade = \"N/A\"" syntax is pseudocode — Grade has int underlying type and cannot hold a string. Putting GradeNA in the same const block before the iota line would shift all existing constant values (GradeA would become 1 instead of 0), breaking gradeStrings and GradeFromScore. A separate const block (const GradeNA Grade = -1) keeps all existing constants at their current values, satisfying "Do not alter any existing grade constants". The "N/A" representation is achieved by adding GradeNA: "N/A" to the gradeStrings map.
