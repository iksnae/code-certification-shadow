**P0 (immediate safety fix):** Don't fabricate grades for unanalyzed languages. When a unit's language has no registered analyzer (`analysis.ForLanguage(lang)==nil`) AND no tool collectors in executor.go, the certifier should mark the unit with a special status (e.g., "unanalyzable") and grade "N/A", exclude from aggregates, and surface a warning. This requires: (a) a central registry of which languages have analysis tiers, (b) a check in the certifier pipeline.

**P1:** Reconcile the 0.5 default with the Score() contract. When policy rules reference metrics that don't exist in evidence, the evaluator creates violations that introduce dimensions at penalized values. The doc comment says absent dimensions don't dilute. Options: (a) make missing-evidence violations NOT introduce dimensions (only penalize existing ones), (b) change the doc, (c) don't create violations for missing evidence when the language has no collector for that evidence type.

**P2:** Add a real Swift tier — tree-sitter-swift or SwiftSyntax for per-unit complexity, fan-in/out, dead-code detection.

**P3:** Wire SwiftLint collector or remove the dead template rule.

## Decision
MECHANISM: The "0.5 default" the issue observes comes from policy violations, not from Score() defaults. When a Swift policy rule like `lint-clean` references metric `lint_errors` but no lint evidence exists, `extractMetric` returns -1, `evaluateRule` returns a violation "missing evidence for metric lint_errors" with the rule's severity. In Score(), violations introduce dimensions at `0.80 - severityPenalty`. For severity=error rules: penalty=0.30 → dimension enters at 0.50. This is why correctness=0.5 and testability=0.5 appear in every Swift record — the swiftStandardPolicy template has lint-clean and test-pass rules both at severity=error.

The contradiction with the Score() doc comment ("Dimensions without evidence are absent — they don't dilute the average") is real: Score() itself doesn't add missing dimensions, but violations from the policy evaluator DO add them before Score() runs. The policy evaluator treats missing evidence as a violation, not as "evidence absent."
