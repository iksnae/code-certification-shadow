# Decision journal — 29


## Decision
The language_tiers package will use analysis.ForLanguage() as the single source of truth for parser availability. If ForLanguage returns a non-nil Analyzer, the language is TierFull; otherwise TierGeneric. This means: go, ts, js, py, rs are TierFull. Everything else (swift, ruby, c, etc.) is TierGeneric. The js registration (init in ts_analyzer.go) means JS also gets TierFull — this is consistent because JS uses the same tree-sitter grammar as TS.

## Decision
Confirmed registered analyzers via init() in analysis package: go (go_analyzer.go:22), ts & js (ts_analyzer.go:19-20), py (python_analyzer.go:16), rs (rust_analyzer.go:15). analysis.ForLanguage calls registry lookup — returns nil for unregistered languages. TierForLanguage will use this: non-nil → TierFull, nil → TierGeneric.
